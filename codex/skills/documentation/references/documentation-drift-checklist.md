# Documentation Drift Checklist

用于判断代码改动后是否需要同步更新文档。

## 何时查看

- 改了 API、schema、配置、CLI、hooks、agents、部署流程
- 改了用户可见行为或关键工作流

## 检查清单

- API 路径、请求参数、响应结构是否变化
- 配置键名、默认值、环境变量是否变化
- CLI 命令、参数、启动方式是否变化
- 部署流程、运行依赖、镜像行为是否变化
- hooks、agents、配置策略是否变化
- 用户可见行为或关键工作流是否变化

## 优先更新

- README
- 运维/部署说明
- API 文档
- 迁移说明
