# Documentation Bootstrap

用于初始化项目最小可用文档骨架。

## 何时查看

- 新仓库缺少 `README.md`、`AGENTS.md`、`docs/` 或 `llmdoc/`
- 需要为后续协作建立最小文档入口

## 最小骨架建议

- `README.md`：项目作用、快速开始、关键命令
- `AGENTS.md`：面向 Codex 的工作约束与项目上下文
- `docs/` 或 `llmdoc/`：只有在项目复杂度需要时再创建

## 适合初始化的内容

- 项目概览
- 运行/开发入口
- 主要目录说明
- 关键依赖或外部系统说明

## 暂时不要初始化的内容

- 细到每个模块的空白模板
- 无明确 owner 的大量 ADR
- 没有真实内容支撑的 API 文档骨架

## 注意事项

- 先保证最小可用入口，再逐步扩展，不要初始化一套没人维护的空骨架
