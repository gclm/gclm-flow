# Handling Review Feedback

用于处理已经出现的 review comment。这里不重复讲 code review 主流程，只保留反馈处理细节。

## 何时查看

- 已收到 reviewer comment
- 评论不够明确，需要先澄清再处理
- 需要决定是接受、部分接受还是技术反驳

## 处理顺序

- 先判断评论指出的是问题，还是单纯偏好
- 检查评论是否和当前需求、平台约束、兼容性要求冲突
- 对模糊评论先澄清，不要只修你猜到的一半
- 外部 reviewer 的建议如果和代码库现实不符，要直接指出原因

## 常见输出

- 已修复：`已修复，变更位于 ...，原因是 ...`
- 需澄清：`我理解 X，但 Y 还不明确，需要确认 ...`
- 技术反驳：`当前实现保留此逻辑是因为 ...；若改成 reviewer 建议，会影响 ...`

## 注意事项

- 不要在没验证前说“完全正确，我马上改”
- 不要因为 reviewer 强势就跳过本地验证
- 不要批量改完再统一看结果，先处理阻塞项
