# Test Level Selection

用于决定优先写哪一层测试。

## 何时查看

- 需求已经明确，但还不确定该从单元、集成还是 E2E 开始
- 想避免一上来就写过重的测试

## 选择顺序

从低到高：
- 单元测试
- 集成测试
- 端到端测试

原则：先用成本最低、反馈最快、最能稳定复现问题的那一层。

## 选择信号

适合单元测试：
- 输入输出明确
- 无真实外部依赖
- 逻辑复杂但边界清晰

适合集成测试：
- 需要真实数据库、文件系统、网络协议适配
- 单元测试无法证明模块间装配是否正确

适合 E2E：
- 价值高的用户路径
- 故障常出在多层协作处
- 需要验证真实交互体验

## 注意事项

- 如果更低层测试已经能稳定证明行为，不要为了“更真实”直接跳到 E2E
