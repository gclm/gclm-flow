# Review Workflow

代码审查工作流，保障代码质量。

## 何时使用

- 提交代码前审查
- 合并 PR 前检查
- 安全审计
- 性能评估

## 工作流程

### 1. 建立上下文

```
阅读 llmdoc/
├── reference/conventions/   # 项目约定
└── architecture/            # 相关架构
```

### 2. 分析变更

- 获取 git diff
- 识别修改范围
- 理解变更目的

### 3. 多维度审查

| 维度 | 检查项 |
|------|--------|
| **代码质量** | 可读性、可维护性、可测试性 |
| **安全性** | 输入验证、认证授权、敏感数据 |
| **性能** | 算法效率、资源管理、查询优化 |
| **规范** | 编码规范、最佳实践、项目约定 |

### 4. 生成报告

```markdown
# 代码审查报告

## 概述
- 审查文件: X 个
- 发现问题: Y 个

## 问题列表

### 🔴 严重 (X 个)
1. **[文件:行号]** 问题描述
   - 建议: ...

### 🟡 警告 (Y 个)
1. ...

### 🔵 建议 (Z 个)
1. ...
```

## 审查维度

### 安全检查
- SQL 注入防护
- XSS 防护
- CSRF 防护
- 敏感数据处理

### 性能检查
- N+1 查询
- 缓存使用
- 资源释放

### 代码质量
- 重复代码
- 复杂度
- 测试覆盖

## 相关命令

- `/gclm:review` - 执行审查
- `/gclm:test` - 运行测试
