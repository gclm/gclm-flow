# Plan Workflow

任务规划工作流，所有开发任务的起点。

## 何时使用

- 开始新功能开发
- 遇到复杂任务需要分解
- 需要设计实现方案

## 工作流程

### 1. 建立上下文

```
阅读 llmdoc/
├── index.md           # 导航
├── overview/          # 项目概述
└── architecture/      # 相关架构
```

### 2. 分析需求

- 理解用户请求
- 识别关键功能
- 确定约束条件
- 提出澄清问题

### 3. 扫描项目

- 检查项目结构
- 识别现有代码模式
- 了解项目约定

### 4. 制定计划

```markdown
# 任务计划

## 目标
[清晰描述任务目标]

## 技术栈
- 语言：[检测到的语言]
- 框架：[检测到的框架]

## 执行步骤

### 步骤 1: [名称]
- **描述**: [详细描述]
- **文件**: [涉及的主要文件]
- **验收标准**: [如何验证完成]

### 步骤 2: ...
```

## 输出格式

```markdown
📊 任务规划

🎯 **目标**: [任务描述]

📋 **步骤**:
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

⚠️ **风险**: [潜在问题]

💡 **建议**: [优化建议]
```

## 相关命令

- `/gclm:plan` - 执行规划
- `/gclm:do` - 执行计划
