# 文档写作规范

## 基本原则

1. **简洁**：每个文档不超过 150 行
2. **明确**：使用清晰的语言，避免歧义
3. **结构化**：使用标题、列表、表格组织内容
4. **可引用**：使用代码引用而非大段代码块

## 格式规范

### 标题

```markdown
# 一级标题（文档标题）

## 二级标题（主要章节）

### 三级标题（子章节）
```

### 代码引用

使用文件路径引用而非大段代码：

```markdown
# 推荐
参见 `src/services/user.service.ts:45-60`

# 不推荐
```typescript
// 大段代码...
```
```

### 链接

使用相对路径：

```markdown
参见 [开发指南](../guides/development.md)
```

### 列表

```markdown
- 无序列表项 1
- 无序列表项 2

1. 有序列表项 1
2. 有序列表项 2
```

### 表格

```markdown
| 列 1 | 列 2 | 列 3 |
|------|------|------|
| 数据 | 数据 | 数据 |
```

## 文件命名

- 使用 kebab-case：`project-overview.md`
- 使用有意义的名称：`api-design.md` 而非 `doc1.md`
- ADR 编号：`ADR-001-use-postgres.md`

## 内容组织

### 概述文档

```markdown
# [主题]

## 简介
[一句话描述]

## 核心内容
[主要内容]

## 相关文档
[链接到相关文档]
```

### 指南文档

```markdown
# [任务名称]

## 前提条件
[需要什么]

## 步骤
1. 第一步
2. 第二步
3. 第三步

## 验证
[如何验证完成]

## 故障排除
[常见问题]
```

### 参考文档

```markdown
# [参考主题]

## 概述
[简要说明]

## 详细信息
[详细内容]

## 示例
[使用示例]
```

## 避免的内容

- 大段代码（使用引用代替）
- 过时的信息（定期更新）
- 模糊的描述（如"可能"、"也许"）
- 重复内容（链接到其他文档）
