# Performance Checklist

用于快速扫描高频性能风险。

## 何时查看

- 本次改动涉及热路径、批处理、查询、并发或大对象处理
- code review 中怀疑存在隐藏性能问题

## 重点检查

- 循环内数据库/API 调用，形成 N+1
- 重复序列化、重复扫描大集合、无界缓存增长
- 未分页查询或一次性加载超大数据
- 热路径缺少缓存、批处理、去重
- 锁持有范围过大，串行化了本可并行的工作
- 测试数据规模太小，掩盖真实复杂度

## 注意事项

- 先确认问题是否真的在性能层，不要把正确性问题误判成性能问题
- 没有证据时，不要只因为“看起来慢”就建议大改架构
