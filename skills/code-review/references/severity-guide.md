# Severity Guide

用于统一 review findings 的严重级别判断。

## 何时查看

- review 中已经发现问题，需要判断阻塞级别
- 需要统一 P0-P3 的用法，避免严重级别漂移

## P0

满足任一条件即可归为 P0：
- 权限绕过、鉴权失效、任意读写、命令执行
- 数据破坏、不可逆丢失、账务/支付错误
- 大面积生产不可用或高概率崩溃

## P1

- 主要功能错误，用户路径被打断
- 回归风险高，且已有代码无法兜底
- 并发、重试、状态机等导致的稳定性问题

## P2

- 设计或实现存在明显缺口，但短期未必立刻出事故
- 测试没有覆盖关键失败路径
- 性能退化、可维护性下降、日志/监控缺失

## P3

- 风格、表达、局部抽象、可读性建议
- 不影响正确性与上线安全

## 使用建议

- 严重级别服务于决策，不是为了“显得问题严重”
- 证据不足时，先降一级并写清假设
