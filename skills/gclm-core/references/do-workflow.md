# Do 工作流

根据计划执行代码实现。

## 触发条件

- plan 工作流完成后
- 用户直接要求执行某个任务

## 工作流程

### 1. 接收计划

- 读取生成的计划
- 理解执行步骤
- 确认验收标准

### 2. 代码实现

调用 `builder` 代理：
- 编写新代码
- 修改现有代码
- 编写测试

### 3. 验证完成

- 运行测试
- 检查编译
- 验证功能

### 4. 质量检查

调用 `reviewer` 代理进行质量检查。

## 输出格式

```markdown
# 执行报告

## 已完成
- [x] 步骤 1: ...
- [x] 步骤 2: ...

## 代码变更
- 新增: ...
- 修改: ...
- 删除: ...

## 测试结果
- 单元测试: X/Y 通过
- 集成测试: X/Y 通过
```
