# Ask 工作流

基于项目上下文回答问题。

## 触发条件

- 用户问"怎么"、"为什么"、"如何"
- 用户要求查询某个信息
- 用户需要技术建议

## 工作流程

### 1. 分析问题意图

- 理解问题类型
- 识别关键概念
- 确定回答范围

### 2. 收集上下文

调用 `investigator` 代理：
- 理解代码结构
- 查找相关代码
- 检索项目文档

### 3. 查询历史

调用 `remember` 代理：
- 搜索历史记忆
- 查找相似问题

### 4. 生成回答

- 提供清晰答案
- 引用相关代码
- 提供参考资料

## 输出格式

```markdown
## 回答

[问题答案]

### 相关代码

// 代码示例

### 参考资料
- [文档链接](path/to/doc.md)
- [历史记忆](memory://xxx)

### 相关问题
- [相似问题 1]
- [相似问题 2]
```

## 常见问题类型

| 类型 | 示例 |
|------|------|
| 代码理解 | "这个函数是做什么的？" |
| 实现方式 | "用户认证是如何实现的？" |
| 最佳实践 | "如何处理异常？" |
| 调试方法 | "如何调试数据库连接问题？" |
