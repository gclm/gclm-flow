# /gclm:do - 执行任务

根据计划执行代码实现。

## 用法

```
/gclm:do [计划文件]
```

## 功能

1. **接收计划**
   - 读取生成的计划
   - 理解执行步骤
   - 确认验收标准

2. **代码实现**
   - 编写新代码
   - 修改现有代码
   - 编写测试

3. **验证完成**
   - 运行测试
   - 检查编译
   - 验证功能

## 工作流程

1. 如果没有计划，先调用 `planner` 代理生成
2. 调用 `builder` 代理执行代码实现
3. 调用 `reviewer` 代理进行质量检查
4. 生成执行报告

## 输出

```markdown
# 执行报告

## 已完成
- [x] 步骤 1: ...
- [x] 步骤 2: ...

## 代码变更
- 新增: ...
- 修改: ...
- 删除: ...

## 测试结果
- 单元测试: X/Y 通过
- 集成测试: X/Y 通过
```

## 示例

```bash
# 执行最近生成的计划
/gclm:do

# 执行指定计划
/gclm:do llmdoc/plans/feature-x.md

# 先规划再执行
/gclm:plan 添加用户登录 | /gclm:do
```

## 相关命令

- `/gclm:plan` - 生成计划
- `/gclm:review` - 审查代码
- `/gclm:test` - 运行测试
