# /gclm:verify - 设计验证

对照设计文档验证实现完整性。

## 用法

```
/gclm:verify [选项]
```

## 功能

1. **读取设计文档**：解析 docs/设计方案.md
2. **检查项目结构**：验证目录和文件是否符合设计
3. **生成验证报告**：列出已完成、缺失、多余的内容

## 工作流程

### 1. 解析设计文档

从设计文档提取预期的项目结构：
- 代理列表
- 命令列表
- 技能目录结构
- 规则目录结构

### 2. 扫描实际结构

检查实际项目中的文件和目录。

### 3. 对比分析

| 状态 | 描述 |
|------|------|
| ✅ 符合 | 与设计一致 |
| ⚠️ 缺失 | 设计中有但实现中没有 |
| ➕ 多余 | 实现中有但设计中没有 |

## 输出格式

```markdown
# 设计验证报告

## 概述
- 符合: X 项
- 缺失: Y 项
- 多余: Z 项
- 符合率: N%

## 详细报告

### Agents（代理）
| 设计 | 实际 | 状态 |
|------|------|------|
| planner.md | ✅ 存在 | ✅ |
| builder.md | ✅ 存在 | ✅ |
| reviewer.md | ✅ 存在 | ✅ |
| investigator.md | ✅ 存在 | ✅ |
| recorder.md | ✅ 存在 | ✅ |
| remember.md | ✅ 存在 | ✅ |

### Commands（命令）
| 设计 | 实际 | 状态 |
|------|------|------|
| init.md | ✅ 存在 | ✅ |
| plan.md | ✅ 存在 | ✅ |
| ... | ... | ... |

### Skills（技能）
| 设计路径 | 实际状态 |
|----------|----------|
| core/plan-workflow/ | ✅ |
| core/review-workflow/ | ✅ |
| patterns/java/ | ✅ |
| database/ | ⚠️ 缺失 |
| devops/ | ⚠️ 缺失 |

### Rules（规则）
| 设计路径 | 实际状态 |
|----------|----------|
| core.md | ✅ |
| languages/*.md | ✅ |
| domains/security.md | ⚠️ 缺失 |
| domains/testing.md | ⚠️ 缺失 |

## 建议
1. [修复建议 1]
2. [修复建议 2]
```

## 选项

- `--fix`: 自动修复可修复的问题（创建缺失目录）
- `--json`: 输出 JSON 格式

## 示例

```bash
# 验证项目结构
/gclm:verify

# 验证并输出 JSON
/gclm:verify --json

# 验证并自动创建缺失目录
/gclm:verify --fix
```

## 相关命令

- `/gclm:init` - 初始化项目
- `/gclm:doc` - 更新设计文档
